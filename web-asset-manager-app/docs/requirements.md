# Requirements (Web Asset Manager)

## Functional
- RQ-001: The system shall provide two top-level screens: Assets and Configurations.
- RQ-002: The Devices screen shall list devices and provide a separate create screen for adding devices.
- RQ-003: The Licenses screen shall list licenses and provide a separate create screen for adding licenses.
- RQ-004: The Assets area shall allow editing and deleting devices and licenses.
- RQ-005: The Devices and Licenses screens shall provide search and sortable list views.
- RQ-004: The Assets screen shall list devices and licenses with search filters and sortable columns.
- RQ-005: The Configurations screen shall display configuration cards on a canvas and a list view.
- RQ-006: The Configurations screen shall allow creating, editing, and deleting configurations.
- RQ-007: The Configurations screen shall allow assigning devices to configurations via drag and drop.
- RQ-008: The Configurations screen shall allow assigning licenses to configurations via drag and drop.
- RQ-011: The system shall provide a configuration detail view showing assigned devices and licenses.
- RQ-008: The system shall include sample device and license data on startup when storage is empty.
- RQ-009: The system shall start without errors and show the UI in a browser.
- RQ-010: Configuration cards shall be draggable within the canvas layout and persist their positions.

## Non-Functional
- RN-001: UI should provide visual feedback during drag and drop.
- RN-002: List/search/sort UI should follow the Redmine style for table controls.
- RN-003: Tests should verify startup, sample data seeding, and major user flows.
